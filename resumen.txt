conocida como LVM2, si bien todo esta configurado y funciona perfecto, hay que
tener presente esto siempre que se hable del disco duro, las particiones pueden
verse en el escritorio, sobre la aplicación conky (los redondos sobre el
escritorio). La informacion es parecida a como se muestra a continuación:

A continuación detallo el fin que le daba a cada partición virtual:

- sda8:    representa la raiz de la instalación gnu/linux.
- sda6:    es la partición donde se aloja el nucleo del sistema operativo, y las
           opciones de arranque del ordenador.
- vg-home: representa la particion donde estan alojados los recursos de los
           usuarios.
- vg-opt:  partición para instalación de programas, que no utilizen la
           paqueteria, un ejemplo de este caso, es el programa netbeans.
- vg-usr:  partición para instalación de programas desde la paqueteria de la
           distribución.
- vg-var:  partición donde se alojan los datos de los programas y servicios que
           posee el ordenador, un ejemplo de este caso, son las aplicaciones
           web.
- vg-distfiles: partición que aloja los paquetes de software a instalarse.
- vg-music:     partición donde catalogaba mi música.
- vg-resources: partición donde pongo cualquier información que sea propia del
                laboratorio de desarrollo, por omisión pongo cosas aqui.

Para seguir mas a fondo los razonamientos por los que decidi utilizar esta
configuración, puede consultarse el siguiente manual:

http://docs.huihoo.com/gentoo/resources/document-listing/lvm2.html

SOBRE EL DESARROLLO WEB Y PHP EN GENERAL (27 de Marzo del 2014)
===============================================================

Desde el día que comenze a experimentar con el extraño servidor en producción
del laboratorio de computo, ademas de aquellos que lo administran, note que lo
mejor era tener una maquina que mejore cualquier cualidad especial del servidor
(para decirlo facil, que sea lo mas parecida en contexto a aquella), asi que
mas o menos esa es la razón por la que uso GNU/Linux.

Bueno comenzare diciendo que aqui no se utiliza el servidor Apache. ¿Porque?,
desde hacia tiempo que experimento con un servidor web que tiene buenas
perspectivas para ser de los mas populares, este servidor se conoce como
Engine X (nginx).

Si bien todo esta correctamente configurado, lo mejor para entender como
desplegar una aplicacion web, es recomendable ver los archivos de configuración
de nginx, alojados en /etc/nginx/.

ADMINISTRACIÓN DE BASES DE DATOS
--------------------------------

El antiguo sistema de apoyo administrativo, esta conformado por 9 bases de datos
en PostgreSQL, una por cada departamento que tiene instalado el sistema.

Para la administración eficaz han sido creados 9 aliases:

    apoyoadm_biologia
    apoyoadm_civil
    apoyoadm_electro
    apoyoadm_fisica
    apoyoadm_industrial
    apoyoadm_infsis
    apoyoadm_matematicas
    apoyoadm_mecanica
    apoyoadm_quimica

Todos estos para acceder a la base de datos correspondiente.

Referente a MariaDB, se han creado varias bases de datos, la mayor parte de
ellas contienen datos veridicos pero no muy importantes, ya que solo son para
pruebas durante el desarrollo de software.

Entre estas podemos destacar:

    - apoyoadm:
        base de datos para el nuevo sistema de apoyo administrativo
        (apoyoadm.local).
    - cidetsi:
        base de datos de prueba para la página de laboratorio de desarrollo.
    - comunicados:
        base de datos de prueba para la página de comunicados y avisos.
    - mysql:
        base de datos interna del servidor.
    - test:
        base de datos que viene con el servidor.
    - vocare:
        base de datos para el sistema de convocatorias (vocare.local).

Para la utilización eficaz de estas bases de datos, tambien se han creado un
conjunto de aliases para el acceso rapido.

    db_admin
        Alias para acceder a MariaDB como administrador.
    db_apoyoadm:
        Alias para acceder a la base de datos apoyoadm, en el sitio en
        producción.
    db_vocare:
        Alias para acceder a la base de datos convocatoria, en el sitio en
        producción.
    db:
        Alias para acceder a una base de datos concreta que se encuentre en la
        misma maquina.
        Ej:
            db apoyoadm     // ingresa a la base de datos apoyoadm en localhost.
            db vocare       // ingresa a la base de datos vocare en localhost.
